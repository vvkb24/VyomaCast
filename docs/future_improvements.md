# Future Improvements & Roadmap

This document outlines structural and functional improvements to enhance the VyomaCast pipeline's performance, resilience, and data quality in production.

---

## 1. Age-Based Article Ingestion Filtering
* **Problem**: When a new feed is subscribed to or the database is cleared, the crawler ingests the entire historic backlog present in the RSS feed (which can include articles published weeks or months ago).
* **Improvement**: Implement an age-threshold filter in `FetcherService`.
  * **Design**: Prior to crawling an article URL, parse the RSS entry's `published` timestamp. If `datetime.now(UTC) - published_at > timedelta(hours=24)`, skip the ingestion entirely.
  * **Benefit**: Ensures the dashboard stays strictly focused on current breaking stories.

## 2. Advanced User-Agent & Proxy Rotation
* **Problem**: Mainstream sites (like *The Guardian*, *Reuters*, *OpenAI*) employ aggressive security rules that block custom bot User-Agents (returning 403 Forbidden) or throttle requests.
* **Improvement**: Integrate a browser-grade User-Agent pool and proxy rotation.
  * **Design**: Maintain a rotating list of standard Chrome, Safari, and Firefox User-Agent headers, and route crawler sockets through a rotating proxy gateway.
  * **Benefit**: Eliminates crawler drops and ensures reliable extraction across all news sites.

## 3. Simulated Time & DB Clock Synchronization
* **Problem**: Using database-side timezone utilities like `func.now()` or `NOW()` causes desynchronization issues in simulated/future testing environments where the application clock differs from the physical host machine clock.
* **Improvement**: Standardize on application-side timestamp parameters.
  * **Design**: Never use SQL-server side `func.now()` inside queries. Always pass a client-side timezone-aware UTC timestamp generated by the application.
  * **Benefit**: Guarantees identical scheduling behavior irrespective of the host OS configuration.

## 4. Adaptive SSL Verification Handling
* **Problem**: When running tests or simulations on machines with offset clocks (e.g. set to the future), SSL certificate checks fail because certificates are evaluated against the wrong date.
* **Improvement**: Make SSL check strictness configurable via environment variables.
  * **Design**: Add a `VYOMACAST_STRICT_SSL` flag in `Settings`. If `False`, configure `aiohttp.TCPConnector(ssl=False)`. If `True` (production), enforce strict verification.
  * **Benefit**: Keeps local development robust and flexible without compromising production security.

## 5. Elastic Pipeline Scaling
* **Problem**: Massive RSS spikes (e.g., during breaking news events) can cause message backlogs in NATS JetStream.
* **Improvement**: Implement auto-scaling workers.
  * **Design**: Deploy workers as Docker container instances inside Kubernetes or ECS, and scale the replica count dynamically based on the queue depth of NATS subjects (e.g. `fetch.>` and `extract.>`).
  * **Benefit**: Keeps latency flat and prevents queue starvation.

## 6. Article Cluster Linking persistence Fix
* **Problem**: When the clustering engine processes an article and binds it to a cluster, the `article.cluster_id` update doesn't persist properly back to the `articles` table in PostgreSQL because the ORM update isn't cascading/saving the modified model instance.
* **Improvement**: Fix the article persistence in `ClusterService._save_transaction`.
  * **Design**: Force an explicit `UPDATE articles SET cluster_id = $1 WHERE id = $2` or verify the SQLAlchemy `session.commit()` recognizes the state change on the retrieved `article` instance.
  * **Benefit**: Ensures the data model retains strict referential integrity and makes external data analysis/export via SQL joins possible.
